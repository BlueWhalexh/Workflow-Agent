后端适配器冒烟

这页跟踪后端接入方如何只消费标准化 SDK 信封，而不理解固定流程、确定性 open-agent 或 OpenAgentGraph 的运行时结果形状。

摘要 关键边界 运行证据 关联页面 待补链接

摘要

后端适配器应该像一个很薄的翻译层：入口只调用 SDK 的 `runAgent()`，输出只映射 `agent-backend-response.v1`。后端 UI、审批、产物和审计链路都不需要知道运行时内部字段。

当前可验证结论是：回答、候选补丁、确认问题、路由预览和流程报告可以统一落到稳定的 `status`、`outputKind`、`displayText`、`artifactRefs` 与工作区写入标记。

候选补丁的 `targetWorkspacePaths` 表示建议写入目标，不表示已经写入。只有 SDK 信封里的 `artifacts.wroteWorkspace` 可以决定工作区写入状态。

关键边界

  • 后端不导入 `src/runtime/*`，不解析 `openAgentGraph` 或流程专属结构。
  • `requiresApproval` 只跟候选补丁和路由预览对齐。
  • `artifactRefs` 只来自 SDK 信封，不包含原始供应商负载。
  • 固定流程预览不能被标记成已执行。
