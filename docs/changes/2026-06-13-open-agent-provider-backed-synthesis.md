# Change: Open Agent Provider-backed Synthesis

## Why

Phase 32 proved that real MiMo can drive `OpenAgentGraph` plan and nextAction. The remaining gap is final output quality: answer, draft, and candidate patch content are still generated by deterministic templates.

That means the graph can decide what to do with a provider, but cannot yet use the provider to produce the actual user-facing knowledge output.

## Decision

Add provider-backed synthesis to `OpenAgentGraph`.

The provider may generate structured answer/draft/candidate content, but deterministic runtime code remains responsible for:

- schema validation;
- grounding validation;
- candidate target validation;
- `publishable: false`;
- content sha;
- fixed workflow handoff;
- artifact writing;
- no workspace write enforcement.

## Safety

- Provider synthesis cannot publish.
- Provider synthesis cannot write raw/schema files.
- Provider output must include grounding refs.
- Raw provider response remains redacted and referenced by path only.
- Fake/fixture runtime keeps deterministic fallback.

## Expected Result

After this phase, a real MiMo `llm-graph` smoke should show at least three provider calls:

1. plan;
2. nextAction;
3. synthesize.

The final answer should be provider-generated and grounded, while the workspace remains unchanged.
