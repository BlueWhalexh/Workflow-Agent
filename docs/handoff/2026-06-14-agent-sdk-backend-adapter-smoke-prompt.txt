新增一个薄 backend adapter，例如 `src/sdk/backend-adapter.ts`。
建议 public API：

```ts
export interface BackendAgentRequest {
  workspaceRoot: string;
  userMessage: string;
  runId?: string;
  mode?: AgentSdkRunMode;
  execute?: boolean;
  autoApprove?: boolean;
  providerRuntime?: AgentSdkProviderRuntimeConfig;
  providerRuntimeDependencies?: AgentSdkProviderRuntimeDependencies;
}

export interface BackendAgentResponse {
  schemaVersion: "agent-backend-response.v1";
  runId: string;
  status: AgentSdkRunStatus;
  outputKind: AgentSdkOutputKind;
  displayText: string | null;
  requiresConfirmation: boolean;
  requiresApproval: boolean;
  artifactRefs: string[];
  wroteWorkspace: boolean;
  targetWorkspacePaths: string[];
  source: AgentSdkRunResult;
}
```

Adapter 规则：
- 内部只调用 `createKnowledgeWorkflowAgent().runAgent()` 或 `runAgent()`。
- 不 import `src/runtime/*`。
- 不解析 `openAgent`、`openAgentGraph`、`workflow` 等旧 runtime result shape。
- `displayText`：
  - answer -> answer text；
  - draft -> draft content；
  - candidate-patch -> candidate patch rationale 或第一段 candidate content 的简短摘要；
  - confirmation -> confirmation questions joined；
  - workflow-report / route-preview / none -> `null` 或稳定短文本，测试中明确期望。
- `requiresConfirmation` 仅对 `outputKind: "confirmation"` 为 true。
- `requiresApproval` 对 `candidate-patch` 和 `route-preview` 为 true；对 answer/draft/confirmation/fixed execution 为 false。
- `artifactRefs` 只包含 SDK envelope 中的 artifact refs：`artifactPath`、`reportPath`、`tracePath`、`rawProviderRefs`；不包含 raw provider payload。
- `wroteWorkspace` 直接来自 `AgentSdkRunResult.artifacts.wroteWorkspace`。
- candidate patch 的 `targetWorkspacePaths` 只表示建议目标，不表示已写入。

注意事项：
- 不做 HTTP server、DB schema、auth、approval UI、真实 backend controller。
- 不新增生产依赖。
- 不改变 `agent-sdk-run.v1` contract。
- 不改变 OpenAgent candidate patch artifact-only 边界。
- 不把 fixed workflow preview 误标成已执行。
- 不把 mock/fake/injected-provider 验证描述成真实 provider。
- 如果需要修改超过 5 个文件，先暂停说明。
